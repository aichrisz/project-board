## Goal

Daily/weekly **steadiness**: review ritual, board keyboard, data hygiene, safer import — not new surface sprawl. Local-first, EN UI, brand **Project Board**.

## Ship (all five)

### 1. Weekly review mode
- New route **`/review`** (or full-page panel from Dashboard + Settings + Layout nav/More).
- Computes from current projects + settings.idleDays + dueSoon helpers:
  - KPI strip: active count, in_progress, overdue, idle, starred, due≤7d
  - Sections:
    - **Overdue** (non-terminal with past deadline)
    - **Due soon** (≤7d)
    - **Stale in progress** (in_progress + idle badge logic)
    - **No steps** (active with 0 steps)
    - **Suggested actions** (3–5 short bullets generated by rules, not LLM): e.g. “Archive N idle”, “Add steps to X”, “Export backup if lastExportAt stale”
  - Actions:
    - Link each project to detail
    - Button **Soft-archive all idle in_progress** (confirm) — only those matching idle rule
    - Button **Open weekly snapshot** (reuse existing)
    - Button **Export JSON**
- Nav: Layout desktop + mobile More / tab if fits; shortcut optional `r` for review (document in KeyboardShortcuts help)
- Empty sections collapse or show “None — nice.”

### 2. Board keyboard a11y
- On `/board`, cards focusable (`tabIndex={0}`), visible focus ring.
- With focus on a card:
  - **← / →** (or **h / l**): move project to previous/next **visible** column status (respect showCompleted column set); update status
  - **↑ / ↓** (or **k / j**): move focus to adjacent card in same column
  - **Enter** / **Space**: navigate to project detail
  - **?** or hint text in subtitle: “Keyboard: ←→ change status, ↑↓ move focus, Enter open”
- Do not steal keys when focus is in input/textarea.
- Announce via `aria-live` polite short message on status change (“Moved to In progress”).
- Keep existing mouse DnD.

### 3. Hygiene tools (Settings panel “Hygiene”)
Pure helpers in `src/lib/hygiene.ts` + Settings UI:
- List counts: empty title? (skip), **no steps** (active), **idle in_progress**, **overdue**, **empty tags** optional
- Actions (each confirm):
  - Soft-archive all **idle** `in_progress` projects
  - Soft-archive all **overdue** non-terminal older than N days optional — **skip** if complex; prefer idle-only bulk + list links
  - Show list of “no steps” with links to detail
- Button “Copy hygiene report” → plain text clipboard summary
- No hard delete bulk.

### 4. Step bulk on project detail
In StepList or detail header when steps.length > 0:
- **Mark all done**
- **Clear all done** (set done false)
- **Remove completed steps** (confirm) — delete steps where done
- Wire through store: `setAllStepsDone(projectId, done: boolean)`, `removeCompletedSteps(projectId)`
- Activity: one event per bulk action (not per step)

### 5. Import preview safety
- Settings Import JSON: **do not** apply immediately.
- Parse file → show modal/panel preview:
  - Project count import vs current
  - New ids / overlapping ids counts
  - Sample titles (first 5)
  - Settings keys that would change
- Buttons: **Cancel** | **Replace all** (current import behavior) | optional **Merge by id** (import projects with new ids only; skip existing ids; merge settings only if user checks “apply settings”)
- Keep `parseImportJson`; add `summarizeImport(current, incoming)` in lib.
